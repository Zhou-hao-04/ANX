# NX HTTP Server 部署说明

## 概述

基于 Siemens NX 2306 二次开发的 HTTP 服务，通过 HTTP 接收外部指令，驱动 NX 引擎执行 3D 模型操作并返回结果。

**架构：**
```
客户端 → HTTP POST → NXServer.dll (.NET C#) → C++ DLL (NXOpen API) → NX 引擎
```

## 文件清单

### 核心部署文件（publish\startup\）

#### 放到 `site\startup\`：

| 文件 | 大小 | 说明 |
|------|------|------|
| `NXServerLoader.dll` | 103 KB | **原生 C++ 加载器。** NX 启动时自动加载此 DLL，然后通过 .NET 托管 API 启动 C# 服务器。必须有 `ufsta` 导出 |
| `NXServer.dll` | 18 KB | **C# HTTP 服务器。** 监听端口 12345，处理 HTTP 请求，调用 C++ DLL 执行 NX 操作 |
| `Newtonsoft.Json.dll` | 712 KB | JSON 序列化依赖库 |

#### 放到 `site\application\`：

| 文件 | 大小 | 说明 |
|------|------|------|
| `HFOpen_mold.dll` | 262 KB | 打开 PRT/STP 文件 |
| `ModelAnalysis.dll` | 282 KB | 模型颜色分析（提取实体颜色索引） |
| `CreateImagesDllNX10.dll` | 144 KB | 七视图截图 |
| `ConvertPrtToStepDll.dll` | 144 KB | PRT → STEP 格式转换 |

### 项目文件（编译用）

| 文件 | 说明 |
|------|------|
| `NXServerHttp\NXServer.sln` | C# 主服务项目（Visual Studio 解决方案） |
| `模型分析\ModelAnalysis.sln` | C++ 模型分析 DLL |
| `下载并打开stp或prt\HFOpen_mold.sln` | C++ 打开文件 DLL |
| `生成七视图\ModelToImagesNX10\ModelToImagesNX10.sln` | C++ 七视图截图 DLL |
| `ConvertPrtToStepDllNX10\ConvertPrtToStepDllNX10.sln` | C++ PRT→STEP 转换 DLL |
| `NXServerHttp\export_nxserver.bat` | 编译后运行此脚本为 NXServer.dll 添加 `ufsta` 导出 |

## 部署步骤

### 1. 复制 DLL 到 NX 目录

打开两个资源管理器窗口：

**源目录：**
```
nxserver-main\nxserver-main\publish\startup\
```

**目标目录：**
```
C:\Program Files\Siemens\NX2306\UGALLIANCE\site\
```

复制操作：

```
复制 NXServer.dll              → site\startup\   （需要管理员权限）
复制 Newtonsoft.Json.dll       → site\startup\   （需要管理员权限）
复制 NXServerLoader.dll        → site\startup\   （需要管理员权限）
复制 HFOpen_mold.dll           → site\application\
复制 ModelAnalysis.dll         → site\application\
复制 CreateImagesDllNX10.dll   → site\application\
复制 ConvertPrtToStepDll.dll   → site\application\
```

复制时提示管理员权限，点 **"继续"**。

### 2. 配置端口权限（一次性）

以 **管理员身份** 打开命令提示符（cmd），执行：

```
netsh http add urlacl url=http://+:12345/ user=everyone
```

### 3. 启动服务

1. 启动 **Siemens NX 2306**
2. NX 会自动加载 `NXServerLoader.dll` → 启动 HTTP 服务
3. 服务监听 `http://+:12345/nxcommand/`

### 4. 验证服务

```powershell
curl.exe -X POST http://localhost:12345/nxcommand/ -H "Content-Type: application/json" -d "{"""Type""":1,"""taskId""":999}" --max-time 5
```

返回 `HTTP/1.1 300 ERROR PARAM!` 即表示服务正常运行。

## API 接口

### 获取当前显示部件的实体颜色

**请求：**
```json
{"Type":2,"taskId":1}
```

**curl 命令：**
```powershell
curl.exe -X POST http://localhost:12345/nxcommand/ -H "Content-Type: application/json" -d "{"""Type""":2,"""taskId""":1}" --max-time 10
```

**响应示例：**
```json
{"Type":2,"taskId":1,"success":true,"color_index":129,"material":""}
```

| 字段 | 说明 |
|------|------|
| `success` | `true` 表示成功获取颜色 |
| `color_index` | NX 颜色索引值 |
| `material` | 材质信息（预留字段） |

**注意：** 使用此功能前，需要在 NX 中**手动打开** PRT 文件。

### 下载并打开文件（预留）

```json
{"Type":1,"taskId":1,"fileUrl":"https://server/file.prt"}
```

## 编译说明

修改代码后需要重新编译并部署。

### C# 项目（NXServer.dll）

1. 在 Visual Studio 中打开 `NXServerHttp\NXServer.sln`
2. 选择 **Debug / x64**
3. `Ctrl + Shift + B` 编译
4. 运行 `NXServerHttp\export_nxserver.bat`（给 NXServer.dll 添加 ufsta 导出）

**注意：** DllExport 步骤的报错（"Multiple resource files not allowed"）可以忽略。编译完成后从 `obj\x64\Debug\` 复制 `NXServer.dll` 即可。

### C++ DLL 项目

直接打开对应的 `.sln`，选 Debug/x64 编译。

### 部署更新

编译后把 DLL 复制到 NX 的 `site\startup\` 和 `site\application\`，然后**重启 NX**。

## 常见问题

### Q: NX 启动时提示"未能加载图像"
A: 检查 `site\startup\` 目录下的 DLL 是否完整。`NXServer.dll` 必须是纯 .NET 版本（从 `obj\x64\Debug\` 复制），不是由 DllExport 处理的版本。

### Q: Type=2 请求超时
A: 确认 PRT 文件已经在 NX 中手动打开。`GetDisplayPartColor` 查询的是当前显示部件，不打开新文件。

### Q: NX Open 调用主线程警告
A: 忽略即可。功能正常，不影响稳定性。

### Q: curl 返回 HTTP 000 / 连接失败
A: 执行以下检查：
1. NX 是否正在运行
2. `netsh http add urlacl url=http://+:12345/ user=everyone` 是否已执行（需要管理员权限）
3. 端口 12345 是否被其他程序占用
